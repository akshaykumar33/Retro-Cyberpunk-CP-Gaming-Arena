The following is a set of guidelines for contributing to Retro Cyberpunk CP Arena. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## Code of Conduct

This project and everyone participating in it is governed by the [Retro Cyberpunk CP Arena Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report unacceptable behavior to [Your Email/Contact].

## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report.
* **Use a clear and descriptive title** for the issue to identify the problem.
* **Describe the exact steps to reproduce the problem** in as much detail as possible.
* **Include screenshots and animated GIFs** which show you following the reproduction steps.

### Suggesting Enhancements

This section guides you through submitting an enhancement suggestion, including completely new features and minor improvements to existing functionality.
* **Use a clear and descriptive title** for the issue to identify the suggestion.
* **Provide a step-by-step description of the suggested enhancement** in as much detail as possible.
* **Explain why this enhancement would be useful** to most Retro Cyberpunk CP Arena users.

### Pull Requests

* Fill in the required template.
* Verify that all status checks are passing after you submit.
* Try to keep your code style consistent with the retro/cyberpunk aesthetic where applicable (e.g., comments, variable naming if that's a style we adopt, though standard clean code is preferred).

## Styleguides

### Git Commit Messages

* Use the present tense ("Add feature" not "Added feature")
* Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
* Limit the first line to 72 characters or less
* Reference issues and pull requests liberally after the first line

### JavaScript/Node.js Styleguide

* We follow standard JavaScript/Node.js best practices.
* Ensure your code is readable and well-documented.
